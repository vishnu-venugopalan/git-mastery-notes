# 	🌿 Branching in Git

## 📌 What is a Branch?
A branch is an independent line of development that allows you to add new features without affecting the basecode.

---

## 🎯 Why Branching is Used
- To develop features separately
- To fix bugs safely
- To avoid breaking the main project

---

## ⚙️ Common Commands
- git branch
- git switch
- git checkout -b

---

## 🔄 Basic Workflow

1. Create a new branch
2. Make changes
3. Commit changes
4. Merge back to main

---

## ⚠️ Common Mistakes

- Working directly on main branch
- Forgetting to switch branches
- Not pulling latest changes

---

## 📌 Example Scenario

A developer creates a branch for adding a new feature, works on it independently, and merges it into the main branch after testing.

---

## 🧠 Key insight

Branching helps developers to work on features safely without impacting the main project, which is crucial in team-based development.
