# 🌐 GitHub Workflow

## 📌 What is GitHub Workflow?

GitHub workflow is a structured process used by developers to work on collaborative projects using branches, commits, and pull requests.

---

## 🎯 Why It is Important

- Enables collaboration between developers
- Prevents conflicts in the main codebase
- Ensures code is reviewed before merging

---

## 🔁 Basic Workflow Steps

1. Clone the repository
2. Create a new branch
3. Make changes
4. Commit changes
5. Push branch to GitHub
6. Create a Pull Request
7. Review and merge

---

## 🌿 Branching in Workflow

Each feature or a fix is developed in a seperate branch to keep the main branch stable.

---

## 🔀 What is a Pull Request (PR)?

A Pull Request is a request to merge changes from one branch into another, usually from a feature branch into the main branch.

---

## 👀 Code Review

Before merging, code is reviewed to:
- Check for errors
- Improve code quality
- Ensure proper functionality

---

## ⚠️ Common mistakes

- Working directly on the main branch
- Not pulling latest changes before starting
- Creating large, unstructured commits

---

## 📌 Example Scenario

A developer clones a project, creates a branch for adding a new feature, make changes, pushes the branch, and creates a pull request for review before merging.

---

## 🧠 Key Insight

The GitHub workflow ensures structured collaboration, reducing errors and maintaining code quality in team environments.

---

## ⚡ Real-World Perspective

In professional  environments, no code is directly pushed to the main branch. All changes go through pull requests and reviews to maintain stability and quality.
